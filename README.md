
본 문서는 리눅스 환경에서 프로세스를 확인하고 제어하는 핵심 명령어인 `top`, `ps`, `jobs`, `kill`에 대해 정리한 과제입니다.

---

##  명령어 요약 비교

| 명령어 | 주요 기능 | 특징 |
| :--- | :--- | :--- |
| **`top`** | 실시간 시스템 리소스 및 프로세스 모니터링 | Windows의 '작업 관리자'와 유사, 실시간 갱신 |
| **`ps`** | 현재 실행 중인 프로세스의 스냅샷 출력 | 실행 시점의 프로세스 상태를 정적으로 확인 |
| **`jobs`** | 현재 셸 세션에서 관리되는 작업 목록 표시 | 백그라운드/포어그라운드 작업 상태 확인 |
| **`kill`** | 특정 프로세스에 시그널을 보내 종료 | PID 또는 작업 번호를 지정하여 프로세스 제어 |

---

##  명령어별 상세 조사

### 1. top (Table of Processes)
시스템의 전반적인 상태와 프로세스 리소스(CPU, 메모리) 사용량을 **실시간으로 화면에 출력**합니다.

* **주요 옵션 및 단축키:**
  * `M`: 메모리 사용량 순으로 정렬
  * `P`: CPU 사용량 순으로 정렬 (기본값)
  * `q`: `top` 화면 빠져나가기
* **사용 예시:**
```bash
top```

### 2. ps (Process Status)
현재 시스템에서 실행 중인 프로세스의 **정적인 상태(스냅샷)**를 보여줍니다.

* **주요 옵션:**
  * `ps -ef`: 시스템의 모든 프로세스를 표준 형식으로 출력
  * `ps aux`: CPU 및 메모리 사용량을 포함하여 모든 사용자의 프로세스 상세 출력
* **사용 예시:**
```bash
ps aux | grep python```

### 3. jobs
현재 로그인된 **터미널(셸 세션)의 백그라운드에서 실행 중인 작업**의 목록과 상태를 보여줍니다.

* **상태 종류:**
  * `Running`: 백그라운드에서 실행 중
  * `Stopped`: 일시 정지됨
* **사용 예시:**
```bash
jobs```

### 4. kill
프로세스에 특정 시그널(Signal)을 보내 **프로세스를 종료**시킵니다. 주로 오작동하거나 멈춘 프로그램을 강제 종료할 때 사용합니다.

* **주요 시그널:**
  * `-15` (SIGTERM): 정상 종료 요청 (프로세스가 스스로 정리하고 종료되도록 함)
  * `-9` (SIGKILL): **강제 종료** (예외 없이 즉시 종료)
* **사용 예시:**
```bash
kill -9 1234  # PID가 1234인 프로세스를 강제 종료
kill %1       # jobs로 확인한 작업 번호 1번을 종료```
